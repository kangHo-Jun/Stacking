# Google Cloud Run 배포 가이드 (오류 해결 포함)

## 1. 사전 준비

### 1-1. Google Cloud 프로젝트 생성
- https://console.cloud.google.com 접속
- 새 프로젝트 생성
- 프로젝트 ID 확인 방법:
  터미널에서: gcloud projects list
  주의: 프로젝트 이름과 ID는 다를 수 있음

### 1-2. Google Cloud CLI 설치 (Mac)
- brew install google-cloud-sdk
- 설치 후 반드시 PATH 추가:
  export PATH=/opt/homebrew/share/google-cloud-sdk/bin:"$PATH"
  영구 적용:
  echo 'export PATH=/opt/homebrew/share/google-cloud-sdk/bin:"$PATH"' >> ~/.zshrc
  source ~/.zshrc
- 확인: gcloud --version

### 1-3. 로그인 및 프로젝트 설정
gcloud auth login
gcloud config set project [프로젝트ID]

### 1-4. API 활성화
gcloud services enable run.googleapis.com
gcloud services enable cloudbuild.googleapis.com

## 2. Google Sheets 연동 설정

### 2-1. 서비스 계정 생성
1. https://console.cloud.google.com 접속
2. API 및 서비스 → 사용자 인증 정보
3. 사용자 인증 정보 만들기 → 서비스 계정
4. 이름 입력 후 완료

### 2-2. JSON 키 파일 다운로드
1. 서비스 계정 클릭
2. 키 탭 → 키 추가 → 새 키 만들기
3. JSON 선택 → 만들기
4. 다운로드된 JSON 파일 경로 기억해둘 것

### 2-3. 구글 시트 공유 설정 (중요!)
- 구글 시트 열기 → 공유 클릭
- JSON 파일 안의 client_email 값을 입력
- 권한: 뷰어
- 주의: 이 단계를 빠뜨리면 PermissionError 발생

### 2-4. Secret Manager에 JSON 저장
gcloud secrets create GOOGLE_CREDENTIALS \
  --data-file='[JSON파일경로]' \
  --project=[프로젝트ID]

### 2-5. Secret 접근 권한 부여 (중요!)
- 아래 명령어 실행:
gcloud secrets add-iam-policy-binding GOOGLE_CREDENTIALS \
  --member="serviceAccount:[프로젝트번호]-compute@developer.gserviceaccount.com" \
  --role="roles/secretmanager.secretAccessor" \
  --project=[프로젝트ID]
- 주의: 이 단계를 빠뜨리면 Permission denied 오류 발생
- 프로젝트 번호 확인: gcloud projects list 의 PROJECT_NUMBER 컬럼

## 3. Dockerfile 작성 시 주의사항

### 3-1. 올바른 Dockerfile
FROM python:3.11-slim

RUN apt-get update && apt-get install -y \
    libpango-1.0-0 \
    libpangoft2-1.0-0 \
    libgdk-pixbuf-2.0-0 \
    libffi-dev \
    shared-mime-info \
    fonts-nanum \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
ENV PYTHONPATH=/app
ENV PORT=8080
CMD ["python3", "src/app.py"]

### 3-2. 주의사항
- libgdk-pixbuf2.0-0 은 잘못된 패키지명
  → 반드시 libgdk-pixbuf-2.0-0 으로 작성
- requirements.txt 에 버전 명시 권장:
  flask==3.0.3
  ortools==9.10.4067
  weasyprint==62.3
  gspread==6.1.2
  google-auth==2.29.0

## 4. 배포 명령어

### 4-1. 최종 배포 명령어
cd '[프로젝트폴더경로]'

gcloud run deploy [서비스명] \
  --source project \
  --region asia-northeast3 \
  --platform managed \
  --allow-unauthenticated \
  --set-env-vars USE_SHEETS=true \
  --set-secrets GOOGLE_CREDENTIALS=GOOGLE_CREDENTIALS:latest

### 4-2. 주의사항
- --set-env-vars 에 JSON 직접 입력 불가
  → 반드시 Secret Manager 사용
- --env-vars-file 옵션은 현재 버전에서 미지원

## 5. 자주 발생하는 오류 및 해결법

| 오류 | 원인 | 해결법 |
|------|------|--------|
| RESOURCES_NOT_FOUND | 프로젝트 ID 오타 | gcloud projects list 로 정확한 ID 확인 |
| Package has no installation candidate | Dockerfile 패키지명 오류 | libgdk-pixbuf2.0-0 → libgdk-pixbuf-2.0-0 |
| Build failed: step exited with non-zero status: 100 | apt-get 패키지 오류 | 빌드 로그를 콘솔에서 확인 |
| PermissionError (구글 시트) | 시트 공유 안됨 | 서비스 계정 이메일로 시트 공유 |
| Permission denied on secret | Secret 접근 권한 없음 | secretAccessor 역할 부여 |
| AttributeError: float has no attribute strip | 구글 시트 숫자값 처리 오류 | _normalize_text()에서 str() 변환 추가 |
| Bad syntax for dict arg | JSON을 --set-env-vars에 직접 입력 | Secret Manager 사용 |

## 6. 빌드 오류 로그 확인 방법
- 브라우저에서 확인:
  https://console.cloud.google.com/cloud-build/builds
- 터미널에서 확인:
  gcloud run services logs read [서비스명] \
    --region asia-northeast3 \
    --project [프로젝트ID] \
    --limit 50

## 7. 배포 완료 후 확인
- 서비스 URL 형식:
  https://[서비스명]-[프로젝트번호].[리전].run.app
- Internal Server Error 발생 시
  반드시 로그 확인 후 오류 해결
