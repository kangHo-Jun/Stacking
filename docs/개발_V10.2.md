```md
# 1. 프로젝트 개요

본 시스템은 건축자재(석고보드 등) 출고 시 **비용 최적화 + 법적 안전성 + 현장 실행 가능성**을 동시에 만족하는 차량 선택 및 적재 자동화 시스템이다.

기존 V10.1 구조를 유지하면서, 본 문서는 특히 **Pre-Check 적재 엔진을 실제 구현 가능한 알고리즘 수준으로 구체화**하는 것을 목표로 한다.

---

# 2. 시스템 전체 흐름

```

사용자 입력 (자재명, 수량, 규격)
→ 단위 환산 (팔레트)
→ Pre-Check (2.5D 레이어 기반 적재)
→ MIP 차량 선택 (OR-Tools)
→ 최종 배치 확정
→ 축중 / COG 계산
→ 혼적 제약 검증
→ 위험도 평가
→ 결과 출력 (상차 지시서 + 매뉴얼)

```

---

# 3. 핵심 알고리즘 개념

## 3.1 2.5D 적재 모델

- X, Y: 바닥 평면 배치
- Z: 동일 규격 수직 스택 (제한적 허용)

즉,
> **3D 문제를 2D 레이어 문제로 단순화한 후, 레이어 단위로 최적화**

---

## 3.2 레이어 기반 구조

```

전체 화물
→ 그룹 분류
→ 레이어 생성
→ 레이어별 배치 (BFD)
→ 적재 순서 구성

```

---

# 4. Pre-Check 알고리즘 명세

## 4.1 전체 흐름

```

입력
→ 화물 분류
→ 우선 배치 대상 분리
→ 적재 공간 분할
→ 레이어 생성
→ 레이어별 BFD 배치

→ 결과 반환

```

# 5. 단계별 알고리즘

## 5.1 화물 분류 (Classification)

각 화물은 아래 기준으로 태깅된다.

| 항목 | 설명 |
|------|------|
| 취약 여부 | 석고보드 등 파손 위험 |
| 중량 등급 | 경량 / 일반 / 중량 |
| 방향 제약 | 회전 가능 여부 |
| 하차 순서 | 선하차 여부 |
| 혼적 그룹 | 혼적 가능 여부 |
| 규격 그룹 | 길이/폭 동일 |

---

## 5.2 우선 배치 대상 처리 (Priority Placement)

다음 화물은 일반 배치 전에 처리한다.

### 대상
- 취약 자재 (석고보드)
- 상단 적재 필요 자재
- 하차 우선 자재
- 혼적 불가 자재
- dead space 유발 자재

### 로직

```

if 취약 자재:
최상단 레이어로 분리

if 혼적 불가:
별도 구획 생성

if 하차 우선:
출입구 근처 배치

if dead space 유발:
별도 공간 먼저 확보

```

---

## 5.3 적재 공간 분할 (Space Partitioning)

트럭 바닥을 동적으로 분할한다.

### 초기 상태
```

[{x:0, y:0, width:TruckW, height:TruckL}]

```

### 배치 후
- 남은 공간을 새로운 공간 리스트로 관리

---

## 5.4 레이어 생성 (Layer Generation)

### 생성 기준
- 동일 길이 / 폭
- 동일 높이
- 동일 취급 등급
- 동일 방향 제약
- 동일 하차 그룹

### 예시

```

Layer_1: 석고보드 900x1800
Layer_2: PF보드
Layer_3: 목재 (회전 가능)

```

---

## 5.5 레이어 내부 배치 (BFD)

각 레이어는 Best-Fit Decreasing 방식 적용

### 절차
1. 면적 기준 내림차순 정렬
2. 빈 공간 리스트 순회
3. 가장 잘 맞는 공간 선택
4. 배치 후 공간 분할

### 의사코드

```

sort(items by area desc)

for item in items:
best_space = find_best_fit(item)

```
if best_space:
    place(item)
    split_space()
else:
    unplaced.append(item)
```

```

---

## 5.6 미배치 화물 처리
```

if 미배치 존재:
if 레이어 추가 가능:

```

# 6. 석고보드 특화 규칙

## 6.1 방향
- 수평 적재만 허용
- 세로 적재 금지

## 6.2 위치

| 조건 | 규칙 |
|------|------|
| 취약 자재 | 최상단 |
| 중량 자재 | 하단 |
| 혼적 | 별도 구획 |

## 6.3 그룹화
- 동일 규격 묶음
- 팔레트 단위 유지

# 8. 차량 선택 (MIP)

Google OR-Tools 기반

### 목적
- 차량 비용 최소화
- 차량 수 최소화

### 제약
- 적재 가능 여부 (Pre-Check 통과)
- 축중 제한

---

# 9. 축중 계산

```

W_axle = Σ (Wi × (L - di) / L)

```

- L: 축간 거리
- di: 화물 위치
- Wi: 중량
```
