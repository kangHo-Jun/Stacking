# WHAT / HOW 문서 — 건축자재 적재 최적화 시스템 V10.1

---

## WHAT (무엇을 만드는가)

| # | 기능 | 설명 |
|---|------|------|
| W1 | 자재 입력 | 사용자가 자재명·수량·규격을 직접 입력 |
| W2 | 단위 환산 | 낱장 → 팔레트 변환, 총 중량·부피 계산 |
| W3 | Pre-Check | 배차 전 2.5D Bin Packing 시뮬레이션, 불가 조기 차단 |
| W4 | 차량 선택 | OR-Tools MIP로 최저 비용 차량 조합 산출 |
| W5 | 적재 배치 | COG + 축중 계산, 혼적 제약 검증 |
| W6 | 위험도 평가 | 7개 항목 레벨 산정 (Safe / Caution / Danger / Critical) |
| W7 | 결과 출력 | 상차 지시서 + 위험도 연동 현장 매뉴얼 자동 생성 |

---

## HOW (어떻게 만드는가)

| # | 기술 선택 | 이유 |
|---|-----------|------|
| H1 | Python 3.10 | OR-Tools 호환, 생태계 |
| H2 | Google OR-Tools | MIP 차량 선택, Apache 2.0 |
| H3 | 2.5D Bin Packing (커스텀) | 바닥 배치 + Stacking 실무 모델 |
| H4 | CLI 입력 (1차) | 빠른 검증, UI는 이후 단계 |
| H5 | JSON 출력 (1차) | 결과 구조 확정 후 리포트 변환 |
| H6 | pytest | 단위·통합 테스트 |

---

## Phase 계획

| Phase | 목표 | 산출물 |
|-------|------|--------|
| **P1** | 자재 입력 + 단위 환산 | `input_parser.py` + 테스트 통과 |
| **P2** | Pre-Check 시뮬레이션 | `bin_packing.py` + 테스트 통과 |
| **P3** | 차량 선택 MIP | `vehicle_selector.py` + 테스트 통과 |
| **P4** | 적재 배치 + 축중 | `loader.py` + 테스트 통과 |
| **P5** | 위험도 평가 | `risk_evaluator.py` + 테스트 통과 |
| **P6** | 결과 출력 | `report_generator.py` + 통합 테스트 통과 |

---

## 고정 검증 기준 (회귀 테스트 — 변경 금지)

| ID | 항목 | 기대값 |
|----|------|--------|
| T01 | 석고보드 50장 → 팔레트 환산 | 2팔레트 (25장/팔레트 기준) |
| T02 | 총 중량 1,200kg → 1톤 차량 선택 불가 | Critical 반환 |
| T03 | 축중 11톤 입력 | Critical 반환 |
| T04 | 전후 편차 35% | 불가(Critical) 반환 |
| T05 | 좌우 편차 15% | 주의(Caution) 반환 |
| T06 | 취약 자재 하단 배치 | 위험(Danger) 반환 |
| T07 | 정상 적재 전체 케이스 | Safe 반환 + 상차 지시서 JSON 생성 |

> 위 7개 테스트는 Phase 전체에서 회귀 유지 필수. 튜닝 전 전원 통과 확인.
