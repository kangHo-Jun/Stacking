## 2. 백엔드 기능 목록

### 2.1 Python 파일별 처리 내용

#### `src/input_parser.py`
- `load_material_db()`
  - `자재정보.csv`를 읽어 딕셔너리로 로드한다.
  - key 형식은 `자재명_규격(mm)_두께(mm)`이다.
- `_calculate_unit_volume_m3()`
  - 규격 파싱이 되면 `가로 x 세로 x 두께`로 낱장 부피를 계산한다.
  - 규격 파싱이 실패하면 기본 박스 규격 `1000 x 1000 x 500(mm)`를 사용한다.
- `process_orders()`
  - 주문별 팔레트 수, 총 중량, 총 부피를 계산한다.
  - 결과에 `handling_grade`, `preferred_position`을 포함한다.

#### `src/bin_packing.py`
- `load_vehicle_db()`
  - `차량정보.csv`를 읽어 차량 리스트를 만든다.
  - 적재함 길이/너비/높이로 `cargo_volume_m3`를 계산한다.
  - 운임이 비어 있으면 `_infer_freight_cost()`로 보간한다.
- `evaluate_vehicle_feasibility()`
  - 총 중량과 총 부피를 차량별로 비교해 적재 가능 여부를 산출한다.
  - 초과 사유를 문자열로 만든다.
- `filter_feasible_vehicles()`
  - `feasible == True` 차량만 반환한다.

#### `src/vehicle_selector.py`
- `_build_pallets()`
  - 주문 결과를 팔레트 또는 낱장 단위 리스트로 변환한다.
- `_allocate_pallets_to_vehicles()`
  - 생성된 적재 단위를 선택 차량 인스턴스들에 분산 배치한다.
- `_select_single_vehicle()`
  - 단일 차량 후보 중 운임 최소 1대를 MIP로 선택한다.
- `_select_multi_vehicle()`
  - 차량별 사용 대수 정수 변수를 만들고 전체 중량을 커버하는 최소 운임 조합을 구한다.
- `select_optimal_vehicle()`
  - `order_items`가 없으면 단일 차량 선택, 있으면 다중 차량 선택을 수행한다.

#### `src/loader.py`
- `_make_pallets()`
  - 주문 항목을 적재 단위로 재구성한다.
- `_plan_loading_from_pallets()`
  - `preferred_position`, `weight_kg`, `unit_type` 순서로 정렬한다.
  - 정해진 `position_cycle`에 따라 `front/rear`, `left/right`, `bottom/top` 배치를 부여한다.
  - 전후/좌우 편차, 상단 비중, 축중, 상단/하단 배치 위반을 계산한다.
- `plan_loading()`
  - 단일 차량 적재 결과를 만든다.
- `plan_fleet_loading()`
  - 차량 조합 전체에 대해 차량별 적재 결과를 만든다.

#### `src/risk_evaluator.py`
- `evaluate_risk()`
  - 단일 차량 적재 결과의 7개 항목 위험도를 계산한다.
- `evaluate_fleet_risk()`
  - 차량별 위험도를 계산한 뒤 전체 최종 위험도를 계산한다.

#### `src/report_generator.py`
- `generate_report()`
  - 단일 차량 기준 `result.json`, `지시서.txt`를 저장한다.
- `generate_fleet_report()`
  - 다중 차량 기준 결과 JSON과 지시서를 저장한다.

#### `src/visualizer.py`
- `_layout_items()`
  - 적재 아이템을 무게 기준으로 정렬하고 시각화 좌표와 레이어를 계산한다.
- `_build_floor_plan_svg()`
  - 평면도 SVG를 만든다.
- `_build_side_view_svg()`
  - 측면도 SVG를 만든다.
- `_build_weight_map_svg()`
  - 4분면 버블맵 SVG를 만든다.
- `build_vehicle_visualization()`
  - 차량 1대의 SVG와 시각화용 item 목록을 묶는다.
- `build_fleet_visualizations()`
  - 차량별 시각화 결과를 dict로 묶는다.

#### `src/app.py`
- `init_history_db()`
  - `history` 테이블을 생성한다.
- `get_material_options()`
  - 자재 dropdown용 key 목록을 반환한다.
- `run_pipeline()`
  - 웹용 전체 처리 파이프라인을 실행한다.
- `save_history()`
  - 실행 결과를 SQLite에 저장한다.
- `build_run_response()`
  - 프론트엔드가 직접 쓰는 JSON 응답을 만든다.
- `download_pdf()`
  - `result.html`을 WeasyPrint로 PDF 변환해 다운로드 응답을 보낸다.

#### `src/main.py`
- `collect_orders()`
  - CLI 입력을 받아 주문 목록을 만든다.
- `run_pipeline()`
  - CLI용 단일 차량 파이프라인을 실행한다.
- `main()`
  - 콘솔 실행 진입점이다.

## 4. 데이터 흐름

### 4.1 웹 실행 흐름

1. 사용자가 `GET /`로 `main.html`에 접속한다.
2. 서버는 `get_material_options()`로 자재 key 목록을 템플릿에 주입한다.
3. 사용자가 자재와 수량을 추가하면 JS가 `state.orders`를 갱신한다.
4. 사용자가 `최적 배차 실행`을 누르면 `FormData`로 `/run`에 POST 요청을 보낸다.
5. `app.py`의 `/run` 라우트가 주문 목록을 파싱한다.
6. `run_pipeline()`이 아래 순서로 실행된다.
   - `load_material_db()`
   - `load_vehicle_db()`
   - `process_orders()`
   - `select_optimal_vehicle()`
   - `plan_fleet_loading()`
   - `evaluate_fleet_risk()`
   - `generate_fleet_report()`
7. 결과 JSON이 `output/result.json`, `output/지시서.txt`에 저장된다.
8. 실행 결과가 `history.db`에 저장된다.
9. `build_run_response()`가 시각화 SVG와 팔레트 목록을 합쳐 JSON 응답을 만든다.
10. 브라우저 JS가 우측 결과 카드와 중앙 시각화, 3D 장면을 다시 렌더링한다.

### 4.2 CLI 실행 흐름

1. 사용자가 `src/main.py`를 실행한다.
2. `collect_orders()`가 자재 key와 수량을 입력받는다.
3. `main.py`의 `run_pipeline()`이 아래 순서로 실행된다.
   - `load_material_db()`
   - `load_vehicle_db()`
   - `process_orders()`
   - `filter_feasible_vehicles()`
   - `select_optimal_vehicle()`
   - `plan_loading()`
   - `evaluate_risk()`
   - `generate_report()`
4. 콘솔에 위험도, 현장매뉴얼, 저장 경로를 출력한다.

### 4.3 입력/처리/출력 요약

| 단계 | 입력 | 처리 | 출력 |
|------|------|------|------|
| 입력 | 자재 key, 수량 | 주문 목록 생성 | `orders` |
| 자재 해석 | `자재정보.csv`, `orders` | 팔레트 수, 중량, 부피 계산 | `order_result` |
| 차량 판정 | `차량정보.csv`, `order_result` | 적재 가능 차량 평가 | 차량 리스트 또는 조합 |
| 차량 선택 | 차량 후보, 주문 중량 | MIP 최적화 | `selection_result` |
| 적재 배치 | 차량 조합, 팔레트 | 위치 배정, 편차/축중 계산 | `load_result` |
| 위험도 | `load_result` | 7개 항목 평가 | `risk_result` |
