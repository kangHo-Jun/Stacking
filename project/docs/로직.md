# 건축자재 적재 최적화 시스템 — 로직 문서

## 1. 시스템 흐름
- P1 `input_parser.py`
  - `자재정보.csv`를 읽어 자재 DB를 딕셔너리로 로드한다.
  - 사용자 주문의 `자재 key + 수량`을 받아 팔레트 수, 총 중량, 총 부피를 계산한다.
- P2 `bin_packing.py`
  - 차량 DB를 로드한다.
  - 주문 총중량, 총부피를 각 차량의 허용 중량/적재함 부피와 비교해 적재 가능 차량을 추린다.
- P3 `vehicle_selector.py`
  - OR-Tools MIP로 전체 중량을 커버하는 최소 운임 차량 조합을 선택한다.
  - 선택된 차량 인스턴스별로 팔레트/낱장을 분배한다.
- P4 `loader.py`
  - 차량별 적재 순서를 만들고 전후/좌우/상단 편차, 축중, 적재위치 위반 여부를 계산한다.
- P5 `risk_evaluator.py`
  - 과적, 축중, 공간, 편차, 상단비중, 적재위치 위반을 기준으로 위험도를 판정한다.
- P6 `report_generator.py`
  - 결과를 `output/result.json`, `output/지시서.txt`로 저장한다.

## 2. 단위 환산 (input_parser.py)
- 자재 key 생성 방식
  - `"{자재명}_{규격(mm)}_{두께(mm)}"`
- 낱장 → 팔레트 변환 공식
  - `pallet_count = ceil(quantity / 팔레트당적재수)`
- 총 중량 계산 방식
  - `item_weight_kg = 낱장무게(kg) * quantity`
- 총 부피 계산 방식
  - `item_volume_m3 = 낱장부피(m3) * quantity`
  - 낱장부피는 `규격(mm)`과 `두께(mm)`를 곱해 `m3`로 환산한다.
- 자재 DB CSV 컬럼
  - `자재명`: 화면 표시명 및 key 구성 요소
  - `규격(mm)`: 예 `900x1800`
  - `두께(mm)`: 판재 두께
  - `낱장무게(kg)`: 1장 기준 무게
  - `팔레트당적재수`: 1팔레트 적재 수량
  - `팔레트무게(kg)`: 팔레트 만재 기준 중량
  - `취급등급`: 위험도/적재 성격 구분용
  - `방향고정`: 방향 제약 메타데이터
  - `적재위치`: `하단` 또는 `상단`

## 3. Pre-Check 시뮬레이션 (bin_packing.py)
- 현재 구현은 엄밀한 2.5D Bin Packing 알고리즘이 아니라, 주문 총량을 차량 용량과 비교하는 사전 필터 단계다.
- 차량 적재 가능 여부 판단 기준
  - 총중량 `<= 차량 최대적재중량(kg)`
  - 총부피 `<= 적재함 부피(m3)`
- 중량 초과 처리
  - `총 중량이 차량 최대적재중량을 초과합니다.` 사유를 기록하고 `feasible=False`
- 공간 초과 처리
  - `총 부피가 적재함 부피를 초과합니다.` 사유를 기록하고 `feasible=False`
- 반환값
  - 가능 여부
  - 사유 문자열
  - 중량비율 `weight_ratio`
  - 부피비율 `volume_ratio`
  - 차량 제원 및 운임
- 운임 공백 처리
  - `운임(원)` 값이 비어 있으면 같은/인접 중량 구간 운임으로 보간 또는 비례 추정한다.
- 처리 한계 기준
  - 현재 코드에 `자재 20종 / 팔레트 50개 / 차량 10대`와 같은 명시적 제한 로직은 없다.
  - 즉, 위 한계는 코드상 강제되지 않는다.

## 4. MIP 차량 선택 (vehicle_selector.py)
- OR-Tools MIP 개념
  - 정수계획으로 차량 종류별 사용 대수를 정수 변수로 두고, 제약을 만족하는 최소 비용 해를 찾는다.
- 목적 함수
  - `sum(차량대수_i * 운임_i)` 최소화
- 제약 조건
  - `sum(차량대수_i * 최대적재중량_i) >= 전체 주문 중량`
- 다중 차량 조합 방식
  - 차량 종류별 정수 변수 `vehicle_count_i`를 만든다.
  - 전체 중량을 커버하도록 하면서 총 운임이 가장 낮은 조합을 고른다.
  - 결과는 `vehicle_counts`, `selected_vehicles`, `vehicle_allocations`로 반환한다.
- 차량별 자재 분배 방식
  - 주문을 팔레트/낱장 단위로 분해한다.
  - 분해된 적재 단위를 중량 기준 내림차순으로 정렬한다.
  - 각 단위를 현재 적재비율이 가장 낮은 차량에 순차 배정한다.
- 단일 차량 선택 모드
  - `order_items` 없이 호출하면 feasible 차량 중 1대를 고르는 단일 MIP를 사용한다.

## 5. 적재 배치 (loader.py)
- 현재 구현된 배치 우선순위
  1. `적재위치=하단` 자재 먼저
  2. 같은 조건이면 무거운 자재 먼저
  3. 같은 조건이면 `pallet`이 `sheet`보다 먼저
- 중요:
  - 요청서에 적힌 `취급등급 C → B → A` 우선순위는 현재 코드에 구현되어 있지 않다.
  - 현재 [loader.py](/Users/zart/Library/Mobile%20Documents/com~apple~CloudDocs/프로젝트/적재시스템/project/src/loader.py)는 `취급등급`을 정렬 우선순위로 사용하지 않는다.
- 적재위치 로직
  - `preferred_position=하단`은 먼저 배치된다.
  - `preferred_position=상단`은 뒤로 배치된다.
  - 상단 자재가 바닥(`vertical_zone=bottom`)에 놓이고 동시에 하단 자재가 상부(`vertical_zone=top`)에 있으면 `top_below_bottom_violation=True`
- 전후 편차 계산
  - `abs(front_weight - rear_weight) / total_weight * 100`
- 좌우 편차 계산
  - `abs(left_weight - right_weight) / total_weight * 100`
- 상단 비중 계산
  - `top_weight / total_weight * 100`
- 축중 계산 방식
  - 차량 길이를 축 수만큼 등분해 축 중심을 잡는다.
  - 각 적재물의 `x_ratio` 기준 위치를 계산한다.
  - 가장 가까운 축에 중량을 더한다.
  - 어느 축이든 `10,000kg` 초과면 `axle_overload_critical=True`

## 6. 위험도 평가 (risk_evaluator.py)
- 레벨 정의
  - `Safe`
  - `Caution`
  - `Danger`
  - `Critical`
- 항목별 판단 기준표

| 항목 | 기준 |
| --- | --- |
| 과적 | `<=85 Safe`, `<95 Caution`, `<=100 Danger`, `>100 Critical` |
| 축중 | 축중 초과 있으면 `Critical`, 아니면 `Safe` |
| 공간 | `<=70 Safe`, `<=95 Caution`, `>95 Danger` |
| 전후편차 | `<=10 Safe`, `<=20 Caution`, `<=30 Danger`, `>30 Critical` |
| 좌우편차 | `<=10 Safe`, `<=20 Caution`, `<=30 Danger`, `>30 Critical` |
| 상단비중 | `<=40 Safe`, `<=60 Caution`, `>60 Danger` |
| 적재위치 | `top_below_bottom_violation=True`면 `Danger`, 아니면 `Safe` |

- 최종 레벨 결정 방식
  - 7개 항목 레벨 중 가장 높은 레벨을 최종 위험도로 사용한다.
- 현장 매뉴얼 연동
  - `Safe`: 표준 결속 및 상차 사진 촬영 후 출고
  - `Caution`: 편차와 공간 사용률 재확인 후 출고
  - `Danger`: 상단 적재 또는 위치 위반 해소 후 재검토
  - `Critical`: 즉시 중단 후 과적/축중/편차 문제 우선 해소

## 7. 자재 분류 기준
- 취급등급 정의
  - 현재 CSV 기준 사용 값은 `A`, `B`
  - `A`: 석고보드 계열
  - `B`: 아이소핑크 계열
  - `C`: 현재 CSV와 코드 우선순위에는 사용되지 않음
- 방향고정 정의
  - `Y`: 방향 고정 필요
  - `N`: 방향 고정 없음
- 적재위치 정의
  - `하단`: 먼저 배치해야 하는 자재
  - `상단`: 나중에 배치해야 하는 자재
- 품목별 분류표
  - 석고보드 전 종류: `A / Y / 하단`
  - 아이소핑크 전 종류: `B / N / 상단`
