# 건축자재 적재 최적화 시스템 — UI 문서 V2 (의사결정 중심)

---

# 1. 설계 목표

기존:
→ 데이터 표시 UI

변경:
→ **의사결정 UI**

핵심:
- 결과를 보여주는 것이 아니라
- **판단 이유 + 행동 선택**을 제공한다

---

# 2. 전체 레이아웃

## 2.1 구조

```text
[Left] 입력
[Center] 시각화 (보조)
[Right] 결과 + 의사결정 (핵심)
```

## 2.2 패널 역할
| 영역 | 역할 |
| :--- | :--- |
| **Left** | 자재 입력 |
| **Center** | 적재 시각화 |
| **Right** | 결과 + 판단 + 선택 |

---

# 3. Left Panel (입력)
*기본적으로 기존 기능 유지*
- 자재 선택
- 수량 입력
- 추가 버튼
- 입력 리스트
- 실행 버튼

---

# 4. Center Panel (시각화)
## 4.1 구성
- 기본: 2.5D 평면도
- 옵션: 3D 보기 (접힘 상태)
- 옵션: 버블맵 (접힘 상태)

## 4.2 원칙
- 시각화는 **“보조”**
- 최종 판단 근거는 Right Panel에서 제공

---

# 5. Right Panel (핵심 UI)
👉 전체 UI의 80% 가치를 담당

## 5.1 구조
1. **요약 (Summary)**
2. **이유 (Explain)**
3. **보정 (Correction)**
4. **상세 (Detail)**
5. **선택 (Action)**

## 5.2 ① 요약 (Summary)
**목적**: 한눈에 상태 판단
**구성**:
- 🚛 추천 차량: 5톤_플러스
- ✔ 출고 가능
- ⚠ 위험도: 주의 (동적 안정성 확보됨)

**규칙**:
- 차량명 강조
- 위험도 색상 표시 (Safe/Caution/Danger)
- 출고 가능 여부 명확 표시

## 5.3 ② 이유 (Explain)
**목적**: 신뢰 확보
**구성**:
📌 **차량 선택 이유**
- 3.5톤 → 적재 불가 (중량/부피 초과)
- 5톤 → 동적 위험 발생 (급제동 시 안전성 미달)
- **→ 5톤_플러스가 가장 안전한 선택**

**규칙**:
- 비교 기반 설명
- 불필요한 수치 제거
- 3줄 이내 요약

## 5.4 ③ 보정 (Correction)
**목적**: 시스템 동작 가시화 및 안전 근거 제시
**구성**:
⚙ **자동 보정 적용됨**
- 전체 화물 후방 이동 +550mm
- 무게 중심 안정화
- 위험도 개선: **Critical → Caution**

**규칙**:
- 변경 전 → 변경 후 상태 표시
- 반드시 “개선 결과” 포함

## 5.5 ④ 상세 (Detail)
**목적**: 필요 시 수치 확인
**구성**:
📦 **적재 상세**
- 레이어 1: 석고보드 900x2400 × 8
- 레이어 2: 석고보드 900x2400 × 2
- 무게 중심: 정적 50.8% / 동적 58.2%

**규칙**:
- 접힘(Collapsible) 가능
- 기본은 최소 표시

## 5.6 ⑤ 선택 (Action) ⭐ 핵심
**목적**: 사용자 최종 의사결정
**구성**:
🛠 **선택**
- **[ 이대로 출고 ]** (Primary Action)
- [ 차량 변경 ]
- [ 분할 배차 ]
- [ 강제 진행 ] (Danger 발생 시 활성화)

**규칙**:
- 반드시 버튼 4개 제공
- Primary Action은 1개만 강조
- [강제 진행]은 경고 색상/조건부 활성화

---

# 6. 위험도 UI 수준 발전
- **기존 문제**: 숫자 중심, 행동 지침 부재
- **변경**: 🚨 급제동 시 앞으로 쏠림 위험
- **조치**: 차량 변경 또는 분할 필요
- **규칙**: 숫자보다는 **행동과 해결 방법** 위주로 텍스트 구성

---

# 7. 자동 보정 시각화 (ASCII/Bar)
Before: `[■■■■■■        ]`
After:  `[   ■■■■■■     ]`
**규칙**: 단순 바(Bar) 형태 사용 (복잡한 그래프 금지)

---

# 8. 핵심 원칙
1. **UI 역할**: 단순히 결과를 보여주는 것이 아니라 **판단을 제공**한다.
2. **사용자 경험**: "왜 이 결과인지 이해하고, 내가 직접 선택한다"는 느낌을 준다.
